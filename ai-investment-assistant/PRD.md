## 1. 概要

### 製品名
**KabuAI** — 個人投資家向け AI 投資アシスタント

### プロダクトビジョン
「投資判断の壁を下げ、個人投資家が自信を持って意思決定できる世界をつくる」

### 解決する課題
| 課題 | 現状の痛み | 頻度 |
|------|-----------|------|
| 情報過多 | 決算・ニュース・指標を自分で整理する時間がない | 毎日 |
| 分析スキル不足 | PER・ROE等の指標の意味・比較方法がわからない | 常時 |
| ポートフォリオ管理 | 複数証券会社に分散していて全体像が見えない | 常時 |
| 感情的判断 | 相場急変時に冷静な判断ができない | 月数回 |

### ターゲットユーザー
**プライマリ:** 投資歴 1〜5 年の個人投資家
- 国内株・米国株・ETF を混合保有
- 証券口座を 1〜3 社に分散
- 月に数回チェック、売買は月 1〜4 回程度
- スマホ・PC 両方で利用

**ユーザーペルソナ**
> 田中 健太（32歳 / 会社員 / 投資歴3年）  
> 月収40万、投資資金300万。NISAで国内株5銘柄、特定口座で米国ETF3本を保有。  
> 毎朝15分だけ株価チェックするが、指標の深い分析をする時間も知識もない。  
> 損益がマイナスになると「売るべきか持ち続けるべきか」で悩み続ける。

## 2. 製品スコープ

### MVP に含むもの（2週間）
- ユーザー認証（Supabase Auth）
- ポートフォリオ登録・管理（銘柄・保有数・取得単価）
- 損益サマリーダッシュボード
- AI チャット（Claude API）― 保有銘柄の情報を文脈に自動渡し
- 株価データ取得（無料 API 連携）

### MVP に含まないもの（v2 以降）
- 自動口座連携（証券会社 API）
- プッシュ通知・アラート
- バックテスト機能
- ソーシャル機能（他ユーザーとの比較）
- 有料プラン・課金機能
- スマホネイティブアプリ

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | ページ初期表示 < 2秒（LCP） |
| セキュリティ | RLS でユーザーデータ完全分離、API キーは環境変数管理 |
| 可用性 | Vercel + Supabase のデフォルト SLA に依存 |
| スケーラビリティ | MVP は最大 100 ユーザー想定 |
| モバイル対応 | レスポンシブ（SP / タブレット / PC）|

## 6. 制約・前提条件

- **株価データ:** 無料枠内（Yahoo Finance API / Alpha Vantage / J-Quants）を利用
- **AI コスト:** Claude API はユーザー数が少ない間は個人負担で運用
- **法的免責:** 投資助言業の登録なし → UI 上に「投資判断はご自身の責任で」と明記必須
- **データ保管:** ポートフォリオデータは Supabase に保管、バックアップは Supabase 自動バックアップに依存
