# PRD — 投資日記アプリ（InvestLog）

**バージョン:** 1.0  
**作成日:** 2026-06-28  
**ステータス:** Draft

---

## 1. 製品概要

### 1.1 プロダクトビジョン

個人投資家が売買記録・購入理由・売却理由・振り返りを一元管理し、AIによるパターン分析で投資判断の質を継続的に改善できるアプリケーション。

### 1.2 解決する課題

| 課題 | 現状 | 解決策 |
|------|------|--------|
| 売買理由を記録していない | スプレッドシートや記憶頼り | 構造化フォームで強制記録 |
| 感情的な売買を繰り返す | 振り返りの機会がない | 振り返り機能 + AIパターン分析 |
| 自分の勝ちパターンを知らない | データが散在 | AI分析で勝率・成功パターン抽出 |
| 株価情報との突き合わせが手間 | 別ツールで確認 | Yahoo Finance連携で自動取得 |

### 1.3 ターゲットユーザー

**プライマリ:** 国内個人投資家（日本株・米国株）  
**スキルレベル:** 初心者〜中級者（スマホ・PCどちらでも操作できる）  
**投資スタイル:** 個別株投資、数週間〜数年の保有期間

## 2. 機能要件

### Phase 1 — 投資日記（MVP）

#### F-01: 認証

| ID | 要件 | 優先度 |
|----|------|--------|
| F-01-1 | メールアドレス + パスワードで登録・ログイン | Must |
| F-01-2 | ログアウト | Must |
| F-01-3 | パスワードリセット（メール送信） | Should |

#### F-02: 売買記録（Trade）

| ID | 要件 | 優先度 |
|----|------|--------|
| F-02-1 | 売買記録の新規作成 | Must |
| F-02-2 | 銘柄コード・銘柄名・市場（JP/US）を入力 | Must |
| F-02-3 | 売買種別（BUY/SELL）を選択 | Must |
| F-02-4 | 数量・単価・取引日を入力 | Must |
| F-02-5 | 購入理由をテキストで記録（BUY時） | Must |
| F-02-6 | 売却理由をテキストで記録（SELL時） | Must |
| F-02-7 | 感情タグを選択（冷静/強欲/恐怖/FOMO/確信） | Should |
| F-02-8 | 売買記録の一覧表示（日付降順） | Must |
| F-02-9 | 売買記録の編集・削除 | Must |

#### F-03: 振り返り（Review）

| ID | 要件 | 優先度 |
|----|------|--------|
| F-03-1 | 任意の売買記録に振り返りを追加 | Must |
| F-03-2 | 結果（利益/損失/トントン）を記録 | Must |
| F-03-3 | 「何がよかったか」「次回改善点」をテキスト入力 | Must |
| F-03-4 | 振り返りスコア（1〜5）で自己評価 | Should |

#### F-04: ダッシュボード

| ID | 要件 | 優先度 |
|----|------|--------|
| F-04-1 | 直近10件の売買記録一覧 | Must |
| F-04-2 | 振り返り未記入の記録をハイライト | Should |
| F-04-3 | 月別売買件数の棒グラフ | Could |

---

### Phase 2 — AI分析（v2）

#### F-05: AI分析レポート

| ID | 要件 | 優先度 |
|----|------|--------|
| F-05-1 | 全売買記録を元に勝率を算出 | Must |
| F-05-2 | 平均利益率・平均損失率を算出 | Must |
| F-05-3 | 成功パターンをAIが自然言語で抽出 | Must |
| F-05-4 | 失敗パターンをAIが自然言語で抽出 | Must |
| F-05-5 | 感情タグ別の勝率比較 | Should |
| F-05-6 | 分析は「既存データの分析のみ」に限定（銘柄推奨禁止） | Must |

> **[制約]** AIは過去データのパターン抽出のみを行う。「〇〇を買うべき」等の銘柄推奨は禁止。

---

### Phase 3 — 株価連携（v3）

#### F-06: 株価取得

| ID | 要件 | 優先度 |
|----|------|--------|
| F-06-1 | Yahoo FinanceからリアルタイムorEOD株価を取得 | Must |
| F-06-2 | PER・PBR・ROEを取得して記録と紐付け | Must |
| F-06-3 | 売買時点での株価を自動記録 | Should |
| F-06-4 | 株価取得失敗時は手動入力にフォールバック | Must |

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンス | ページロード < 2秒（LCP） |
| 可用性 | 99%以上（Vercelホスティング） |
| セキュリティ | RLSによるユーザー間データ完全分離 |
| レスポンシブ | スマホ・PC両対応（min-width: 375px） |
| 認証 | Supabase Auth（セッション管理） |
| データ | ユーザー削除時に全データをカスケード削除 |

## 6. 制約・前提

- 投資助言は行わない（免責表示を全画面フッターに表示）
- AIは既存データのパターン抽出のみ。銘柄推奨禁止。
- 株価データは参考情報。正確性を保証しない旨を明示。
- 個人開発・1名運用を前提としたシンプルな構成。

---

## 7. KPI（MVP判定基準）

```
✅ メール認証でログイン・ログアウトできる
✅ 売買記録を作成・編集・削除できる（購入理由・売却理由含む）
✅ 振り返りを記録できる
✅ ダッシュボードで直近記録が確認できる
✅ スマホで操作できる
✅ 免責表示がフッターに表示されている
```
